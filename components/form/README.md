# フォーム実装について

## 概要
HTML と JavaScript を用いて、バリデーション機能付きフォームを実装しました。  
ブラウザ標準のバリデーションに依存せず、JavaScript による独自判定と  
ユーザー体験を意識したエラー・成功表示の制御を行っています。

---

## 実装内容

### フォームの基本構造

- `label` と `input` を用いたフォームを実装
- HTML の基本的なフォーム構造を理解
- `novalidate` 属性を使用し、ブラウザ標準のバリデーションを無効化

---

### バリデーション処理

- JavaScript による独自バリデーション処理を実装
- 入力内容の妥当性を判定し、エラー時にメッセージを表示
- バリデーションの判定処理と UI 制御を分離
- 複数回使用する処理を関数化し、可読性・保守性を向上

---

### UI / UX の制御

- フォーム送信時、すべての入力内容が正しい場合に success メッセージを表示
- `setTimeout` を使用し、3秒後に success メッセージを自動で非表示
- フォーム送信後に送信ボタンを無効化し、二重送信を防止
- エラー表示後、入力内容が正しくなったタイミングでリアルタイムにエラーを非表示

---

### イベント設計

- `submit` イベントで全体の入力チェックを実行
- `input` イベントでリアルタイムバリデーションを実装
- イベントごとに処理を分離し、責務が分かりやすい構成に整理

---

### その他

- フォーム送信後に `reset()` を使用して入力内容を初期化
- 拡張や再利用を想定した構成を意識して実装
